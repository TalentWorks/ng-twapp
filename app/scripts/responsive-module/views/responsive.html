An excerpt from the Rebellion UI Architecture Blueprint on Responsive Design

3.5.2      Responsive framework

Responsive design is a Web design and development methodology that helps ensure the user interface delivers an optimal viewing experience on any screen size using a fluid grid-based layout. Modern Web applications are often written for consumption via traditional desktops, mobile devices, and tablets. Responsive design is a methodology by which a single codebase can be utilized to deliver an optimal experience to each of these devices. Responsive frameworks have the following primary components:

·         Grid System – design concepts that organize all content into structured components.

·         CSS3 Media Queries - CSS code that allows rendering based on certain dynamic attributes of a page. CSS3 Media Queries allow browsers to detect screen resolution and react when certain breakpoints are reached.

·         Fluid Components – the width of the components is scaled relative to the window.

·         Progressive Enhancement – UI is designed with mobile first in mind and as the view port increases the UI is enhanced accordingly.

Responsive frameworks provide a grid system that divides the viewport size into multiple columns. The predefined CSS classes are then used to create the; page layout specifying the number of columns a component spans in a row. Responsive frameworks usually recognize a number (3-4, e.g.: small, medium, large, extra-large) of screen sizes that each have their own media query. The layout of the page changes on media query breakpoints. Moreover the width of components between two media queries adjusts relative to the window. Layouts defined for the small devices are progressively enhanced as the view port increases e.g.: collapsed layout changes to the horizontal.

Responsive frameworks provide also a set of predefined classes for global styling of standard components that are consistent across different browsers and pre-built UI widgets that accelerate modern UI development (e.g.: dropdowns, button groups, pagination, off canvas navigation, breadcrumbs, etc.). Despite of the amount of predefined components, responsive frameworks still allow full customization to suit the needs of actual project – grid system, media queries breakpoints, colors and styling of components, required plugins, etc.

The benefits of responsive design include:

·         Cost savings: invest in developing a single Web UI that works across multiple devices and screen sizes, rather than separate mobile and desktop Web UI development efforts.

·         Single Codebase: build a site once, reach customers on multiple form factors simultaneously.

·         Consistent Branding: there is only a single reflection of the brand, not multiple, often diverging experiences.

3.5.2.1  Twitter Bootstrap

The Rebellion project will utilize the Twitter Bootstrap framework for development in Less. The framework itself encourages a pattern of development that leverages responsive design principles to reduce extra development effort. By taking a Mobile-first approach to the development of the application, the design is built to be friendly for all user experiences. The bootstrap framework itself offers four different user experiences when integrated into the application:

·         Mobile size (Less than 768px)

·         Table size (Larger than 768px)

·         Desktop size (Larger than 992px)

·         Large size (Larger than 1200px)
